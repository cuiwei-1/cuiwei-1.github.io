JDBC 知识点总结

JDBC（Java Database Connectivity）是一种用于执行 SQL 语句的 Java API，可以为多种关系数据库提供统一访问。以下是对 JDBC 知识点的总结。

一、JDBC 基本概念

  1. 驱动程序（Driver）：JDBC 驱动程序是实现了 JDBC API 的软件组件，它负责与特定的数据库进行通信。不同的数据库厂商提供不同的 JDBC 驱动程序。
  2. 连接（Connection）：通过 JDBC 驱动程序建立与数据库的连接。连接是执行 SQL 语句的基础。
  3. 语句（Statement）：用于向数据库发送 SQL 语句。JDBC 提供了三种类型的语句：Statement、PreparedStatement 和 CallableStatement。
  4. 结果集（ResultSet）：执行查询语句后返回的结果集，包含查询结果的行和列。

二、使用 JDBC 的步骤

  1. 加载驱动程序：
  • 使用 Class.forName() 方法加载数据库驱动程序。例如，对于 MySQL 数据库，可以使用 Class.forName(“com.mysql.cj.jdbc.Driver”) 。
  1. 建立连接：
  • 使用 DriverManager.getConnection() 方法建立与数据库的连接。需要提供数据库的 URL、用户名和密码。例如，对于 MySQL 数据库，连接字符串可能是 jdbc:mysql://localhost:3306/database_name?user=username&password=password 。
  1. 创建语句：
  • 根据需要创建 Statement、PreparedStatement 或 CallableStatement 对象。
  • Statement 用于执行简单的 SQL 语句。
  • PreparedStatement 用于执行预编译的 SQL 语句，可以防止 SQL 注入攻击。
  • CallableStatement 用于执行存储过程。
  1. 执行 SQL 语句：
  • 根据语句类型执行相应的 SQL 语句。
  • 对于查询语句，使用 executeQuery() 方法返回 ResultSet 对象。
  • 对于插入、更新和删除语句，使用 executeUpdate() 方法返回受影响的行数。
  1. 处理结果集：
  • 如果执行的是查询语句，遍历 ResultSet 对象获取查询结果。
  • 可以使用 getXXX() 方法获取结果集中的列值，其中 XXX 表示数据类型。
  1. 关闭资源：
  • 在使用完 JDBC 资源后，务必关闭连接、语句和结果集，以释放资源。

三、PreparedStatement 的优势

  1. 防止 SQL 注入攻击：通过预编译 SQL 语句，将参数与 SQL 语句分离，避免了用户输入被恶意拼接成 SQL 语句的风险。
  2. 提高性能：预编译的 SQL 语句可以被数据库缓存，多次执行时可以提高性能。
  3. 方便参数设置：可以使用 setXXX() 方法设置参数值，其中 XXX 表示数据类型。参数的位置用问号（?）表示。

四、事务处理

  1. JDBC 支持事务处理，可以将多个 SQL 语句作为一个事务执行，保证数据的一致性。
  2. 使用 Connection.setAutoCommit(false) 开启事务，然后在执行完一系列 SQL 语句后，使用 Connection.commit() 提交事务，或者使用 Connection.rollback() 回滚事务。

五、连接池

  1. 为了提高数据库连接的性能，可以使用连接池管理数据库连接。
  2. 连接池预先创建一定数量的连接，当需要连接时从池中获取，使用完毕后归还到池中，而不是每次都创建和销毁连接。
  3. 常见的连接池有 C3P0、DBCP 和 HikariCP 等。

总之，JDBC 是 Java 中访问数据库的重要技术，掌握 JDBC 的基本概念和使用方法对于开发数据库应用程序非常重要。同时，合理使用 PreparedStatement、事务处理和连接池等技术可以提高应用程序的性能和安全性。

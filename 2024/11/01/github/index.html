GitHub 是一个面向开源及私有软件项目的托管平台，它为开发者提供了许多强大的功能。以下是对 GitHub 使用方法的知识点总结。

一、注册与登录

  1. 访问 GitHub 官网，点击“Sign up”进行注册。
  2. 填写用户名、邮箱地址和密码等信息，完成注册。
  3. 注册成功后，使用用户名和密码登录 GitHub。

二、创建仓库（Repository）

  1. 登录后，点击页面右上角的“+”号，选择“New repository”。
  2. 为仓库命名，添加描述（可选），选择仓库的可见性（公开或私有）。
  3. 可以选择初始化仓库，添加 README 文件、添加.gitignore 文件等。
  4. 点击“Create repository”完成仓库创建。

三、克隆仓库

  1. 在仓库页面找到“Clone or download”按钮，点击后会显示仓库的 URL。
  2. 打开命令行工具，使用 git clone 命令加上仓库 URL，将仓库克隆到本地。

例如： git clone https://github.com/username/repository-name.git

四、提交代码

  1. 在本地仓库中进行代码修改、添加新文件等操作。
  2. 使用 git add 命令将修改的文件添加到暂存区。
  • 例如： git add filename 可以添加单个文件； git add. 可以添加当前目录下的所有修改。
  1. 使用 git commit 命令提交修改，添加提交信息。
  • 例如： git commit -m “提交信息描述”
  1. 使用 git push 命令将本地提交推送到远程仓库。

五、分支管理

  1. 创建分支：使用 git branch branch-name 命令创建新分支。
  • 例如： git branch new-feature
  1. 切换分支：使用 git checkout branch-name 命令切换到指定分支。
  • 例如： git checkout new-feature
  1. 合并分支：在一个分支上使用 git merge branch-name 命令将另一个分支的修改合并到当前分支。
  • 例如：在主分支上执行 git merge new-feature 可以将新功能分支的修改合并到主分支。

六、拉取请求（Pull Request）

  1. 当在一个分支上进行了修改并希望将这些修改合并到主分支时，可以创建拉取请求。
  2. 在 GitHub 仓库页面，点击“Pull requests”，然后点击“New pull request”。
  3. 选择要合并的分支和目标分支，查看修改的差异。
  4. 填写拉取请求的描述信息，然后点击“Create pull request”。
  5. 其他团队成员可以对拉取请求进行审查和讨论，确认无误后可以合并。

七、问题跟踪（Issues）

  1. 在仓库页面点击“Issues”可以创建和管理问题。
  2. 可以描述问题、添加标签、分配给特定的人员等。
  3. 开发者可以针对问题进行讨论和回复，直到问题解决。

八、协作与团队管理

  1. 可以邀请其他开发者加入仓库，共同进行开发。
  2. 设置团队成员的权限，如读取、写入、管理员等。
  3. 使用 GitHub 的组织功能可以管理多个仓库和团队。

九、GitHub Actions

  1. GitHub Actions 是一个持续集成和持续部署（CI/CD）平台。
  2. 可以在仓库中创建工作流，定义自动化任务，如代码构建、测试、部署等。
  3. 通过配置 YAML 文件来定义工作流的步骤和触发条件。

总之，GitHub 是一个功能强大的代码托管和协作平台，掌握其使用方法可以提高开发效率，促进团队协作。不断探索和实践，你会发现更多有用的功能和技巧。
